# Cursor Browser Research Protocol - MVR Twin

Use this when Cursor has browser or MCP browser tools available. It is a research
adapter, not an authority source; the kernel still wins market claims.

Before a Build Charter quotes any named incumbent, regulation, license, capital
number, school/clinic/store density, or official approval path:

1. Open the primary source or the most authoritative public source available.
2. Capture the URL, publisher/authority, access date, and the exact claim being supported.
3. If a site requires interaction, use the browser tool and note the observed path.
4. Save screenshots or exported snippets only when legally and operationally appropriate.
5. Add one Source Ledger row per claim: `claim | source | URL | date | status`.
6. Mark unsupported facts `UNKNOWN - not verified`; unknown facts never justify a redirect and never unblock claims.

For regulated claims, prefer regulator, statute, gazette, official license list,
central-bank, ministry, or court sources over blogs. Practitioner guides can
explain the path, but not replace the official source.

For Cursor users: if browser tooling is not available, do not invent evidence.
Write the evidence bill and keep the charter at internal_planning until the
source ledger is real.
